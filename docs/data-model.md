# Code Smart — Data Model

## 1. Design Principles

All data lives in JSON files checked into git. No database, no ORM, no runtime DB connections.

Three data categories:

### Canonical Provider Data (`src/data/providers/*.json`)
One file per provider. Contains provider metadata, plans, models, features, pricing, and usage limits. This is the source of truth for all user-facing plan, feature, and pricing information. Edited manually when providers change their offerings. Validated by Zod at build time.

### AA Index Snapshots (`src/data/aa-snapshots/YYYY-MM-DD.json`)
Weekly snapshots of Artificial Analysis intelligence, coding, and agentic indices per model. Written by GitHub Actions every Monday. **Append-only — never edit or delete old snapshot files.** The history of these files is the 12-week sparkline data source.

### Computed Scores (`src/data/computed-scores.json`)
Pre-computed value scores regenerated at each build from the above two sources by `scripts/recompute-scores.ts`. **Never edit manually.** The `computed_at` timestamp in the file is the authoritative freshness indicator.

## 2. Existing TypeScript Types (unchanged)

All existing types in `src/types/index.ts` are preserved without modification:

| Type | Kind | Description |
|------|------|-------------|
| `Confidence` | union | `"observed" \| "inferred" \| "assumed" \| "stale" \| "unknown"` — provenance badge levels |
| `Provenance` | interface | `{ confidence: Confidence; source_url?: string; last_updated: string; notes?: string }` — attached to any data point that needs a source citation |
| `LimitType` | union | `"token" \| "message" \| "credit" \| "request" \| "unknown"` — the unit type of a usage limit |
| `UsageLimit` | interface | `{ type: LimitType; value: number \| null; unit: string; window: string; provenance: Provenance }` — a single usage limit (e.g., 1M tokens/month) |
| `BenchmarkName` | type | String union of tracked benchmark names |
| `BenchmarkScore` | interface | `{ name: BenchmarkName; score: number; max_score: number; provenance: Provenance }` — a single benchmark result |
| `ModelRef` | interface | `{ model_id: string; access_type: "full" \| "limited" \| "preview" }` — reference to a model from within a plan |
| `Model` | interface | Full model definition: id, name, provider_id, release_date, context_window, benchmarks, provenance |
| `PlanTier` | union | `"free" \| "low" \| "mid" \| "high"` — computed from effective monthly price |
| `ProviderCategory` | union | `"coding-assistant" \| "chat" \| "api" \| "ide-plugin"` — provider type classification |
| `PlanPricing` | interface | monthly_price, annual_price, annual_monthly_effective, billing_interval, currency, provenance |
| `PlanFeatures` | interface | Named boolean or string features: code_completion, chat, agent_mode, ide_integrations, etc. |
| `Plan` | interface | Full plan definition: id, provider_id, name, tier, pricing, models, limits, features, provenance |
| `Provider` | interface | Full provider definition: id, name, logo_url, website_url, category, models[], plans[], provenance |
| `ValueScore` | interface | Will be extended in §3 — existing fields retained for backward compatibility |
| `ComparisonFilter` | interface | User-selected filter state: tiers, models, max_price, billing_interval |

## 3. New TypeScript Types to Add

Add the following to `src/types/index.ts`. These are additive — no existing types are modified or removed.

```typescript
// Snapshot of Artificial Analysis indices for one model at one point in time.
// Stored in src/data/aa-snapshots/YYYY-MM-DD.json (array of these per file).
export interface AAIndexSnapshot {
  model_id: string;                   // matches Model.id in provider JSON
  provider_id: string;                // developer of the model (not reseller)
  fetched_date: string;               // YYYY-MM-DD (matches snapshot filename)
  intelligence_index: number | null;  // AA composite quality score, 0–100
  coding_index: number | null;        // AA coding-specific score, 0–100
  agentic_index: number | null;       // AA agentic tasks score, 0–100
  speed_tps: number | null;           // output tokens per second
  source_url: string;                 // URL of the AA model page fetched
  confidence: Confidence;             // "observed" = direct API fetch;
                                      // "inferred" = proxy inheritance;
                                      // "assumed" = sourced from override file
  methodology_note?: string;          // AA methodology version, if provided in response
  inherits_from?: string;             // model_id of parent model, if this is a proxy
                                      // (e.g., cursor-claude-sonnet inherits from claude-sonnet-4-6)
}

// Token usage estimate for a plan, derived from its published usage limits.
// Generated by recompute-scores.ts; embedded in computed-scores.json.
export interface UsageEstimate {
  plan_id: string;
  limit_basis: LimitType;             // which UsageLimit type was used to compute this
  tokens_per_5h_session: number | null;
  tokens_per_day: number | null;
  tokens_per_week: number | null;     // assumes 5 working days
  tokens_per_month: number | null;    // assumes 20 working days
  assumption_notes: string[];         // human-readable chain of assumptions used
                                      // e.g., ["Assumed 1500 tokens/message (input)",
                                      //        "Assumed 800 tokens/message (output)",
                                      //        "Assumed 40 messages/5h session"]
  confidence: Confidence;             // "observed" only if all inputs were observed;
                                      // otherwise the worst confidence of any input
}

// Extended ValueScore (additive — all existing fields retained for backward compatibility).
// Extended fields are NEW and marked as such.
export interface ValueScore {
  plan_id: string;
  model_id: string;                             // NEW: which model this score is for
                                                // (one ValueScore per plan+model pair)

  // ── New formula components ────────────────────────────────────────────
  weighted_model_quality: number | null;        // NEW: 0–100
                                                // = 0.50×agentic + 0.40×coding + 0.10×speed_score
                                                // null if any required AA index is null
  estimated_monthly_tokens: number | null;      // NEW: from UsageEstimate.tokens_per_month
  quality_adjusted_monthly_usage: number | null; // NEW: = estimated_monthly_tokens × (WMQ / 100)
  value_score_raw: number | null;               // NEW: = QAMU / effective_monthly_price_usd
                                                // null if QAMU is null or price is 0
  value_score_normalized: number | null;        // NEW: 0–100, normalized within price tier
                                                // null if value_score_raw is null
  uncertainty_score: number;                    // NEW: 0–100, see methodology.md
                                                // 0 = all inputs observed, 100 = all unknown

  // ── Existing fields (retained unchanged) ─────────────────────────────
  effective_cost_per_message_usd: number | null;
  benchmark_quality_index: number | null;
  feature_completeness_score: number;
  overall_value_score: number;                  // = value_score_normalized ?? 0
                                                // mapped to new formula; legacy callers unchanged

  score_breakdown: {
    wmq_components: {                           // NEW nested object
      agentic_index: number | null;
      coding_index: number | null;
      speed_score: number | null;               // speed_tps normalized to 0–100 within model set
    };
    weights: {                                  // NEW nested object (constant values)
      agentic: 0.50;
      coding: 0.40;
      speed: 0.10;
    };
    // Legacy fields (retained for backward compatibility)
    cost_score: number;
    benchmark_score: number;
    feature_score: number;
  };
  notes: string[];
  computed_at: string;                          // ISO 8601 timestamp of score computation
}

// Sort key for the model comparison table (used by /api/models query param).
export type ModelSortKey =
  | "intelligence_index"
  | "coding_index"
  | "agentic_index"
  | "speed_tps"
  | "model_age"
  | "value_score";
```

## 4. New JSON File Formats

### `src/data/aa-snapshots/YYYY-MM-DD.json`
One file per weekly AA fetch. Contains `AAIndexSnapshot[]` — one entry per tracked model. Includes both directly-fetched models and proxy-inherited models.

```json
[
  {
    "model_id": "claude-sonnet-4-6",
    "provider_id": "anthropic",
    "fetched_date": "2026-06-16",
    "intelligence_index": 87.3,
    "coding_index": 84.1,
    "agentic_index": 79.6,
    "speed_tps": 95.2,
    "source_url": "https://artificialanalysis.ai/models/claude-3-7-sonnet",
    "confidence": "observed"
  },
  {
    "model_id": "cursor-claude-sonnet",
    "provider_id": "cursor",
    "fetched_date": "2026-06-16",
    "intelligence_index": 87.3,
    "coding_index": 84.1,
    "agentic_index": 79.6,
    "speed_tps": 95.2,
    "source_url": "https://artificialanalysis.ai/models/claude-3-7-sonnet",
    "confidence": "inferred",
    "inherits_from": "claude-sonnet-4-6"
  },
  {
    "model_id": "opencode-byok",
    "provider_id": "opencode",
    "fetched_date": "2026-06-16",
    "intelligence_index": null,
    "coding_index": null,
    "agentic_index": null,
    "speed_tps": null,
    "source_url": "",
    "confidence": "unknown"
  }
]
```

**Rules:**
- If a model has no AA profile and no `inherits_from`, all index fields are `null` and confidence is `"unknown"`.
- If a model inherits from a parent, all index values are copied from the parent's fetched values and confidence is `"inferred"`.
- Only single-level inheritance is supported (no chaining).
- The `fetched_date` matches the filename's date in all rows.

### `src/data/computed-scores.json`
Regenerated by `scripts/recompute-scores.ts` at each build. Contains one `ValueScore` per plan+model pair. This file is the sole source for ranking displays.

```json
{
  "computed_at": "2026-06-16T08:15:00Z",
  "aa_snapshot_date": "2026-06-16",
  "formula_version": "1.0",
  "scores": [
    {
      "plan_id": "anthropic-claude-pro",
      "model_id": "claude-sonnet-4-6",
      "tier": "low",
      "weighted_model_quality": 82.4,
      "estimated_monthly_tokens": 180000,
      "quality_adjusted_monthly_usage": 148320,
      "value_score_raw": 7416.0,
      "value_score_normalized": 78.5,
      "uncertainty_score": 15,
      "effective_cost_per_message_usd": null,
      "benchmark_quality_index": 82.4,
      "feature_completeness_score": 72,
      "overall_value_score": 78,
      "score_breakdown": {
        "wmq_components": {
          "agentic_index": 79.6,
          "coding_index": 84.1,
          "speed_score": 68.4
        },
        "weights": {
          "agentic": 0.50,
          "coding": 0.40,
          "speed": 0.10
        },
        "cost_score": 65,
        "benchmark_score": 82,
        "feature_score": 72
      },
      "notes": [
        "Usage estimated: limit type unknown, assumed 9000 tokens/day × 20 working days",
        "Pricing observed 2026-06-14 from anthropic.com/claude/pricing",
        "Speed score normalized within 25-model set: speed_tps=95.2 → 68.4/100"

```

**Notes:**
- `formula_version` is a string that increments when the scoring formula changes. Old snapshot files are not retroactively rescored — the version records which formula produced the scores.
- `tier` is copied from the plan definition, not recomputed here.
- `overall_value_score` is the integer-rounded `value_score_normalized` for backward compatibility with any code that reads the existing field.

### `src/data/aa-indices-override.json`
Manually maintained fallback used when the AA API fetch fails. Same array structure as snapshot files. Confidence is always `"assumed"` for every record in this file.

```json
[
  {
    "model_id": "claude-sonnet-4-6",
    "provider_id": "anthropic",
    "fetched_date": "2026-06-09",
    "intelligence_index": 87.3,
    "coding_index": 84.1,
    "agentic_index": 79.6,
    "speed_tps": 95.2,
    "source_url": "https://artificialanalysis.ai/models/claude-3-7-sonnet",
    "confidence": "assumed"
  }
]
```

**Maintenance rule:** This file must be updated manually whenever a significant model release changes AA index values materially. It should never be more than 30 days old. The `fetch-aa-indices.ts` script logs a warning if the override file's `fetched_date` is more than 30 days in the past.

---

### `src/data/aa-model-slugs.json`
Maps each `model_id` (as used in provider JSON and snapshot files) to the AA API's naming convention.

```json
{
  "claude-sonnet-4-6": {
    "aa_slug": "claude-3-7-sonnet",
    "aa_url": "https://artificialanalysis.ai/models/claude-3-7-sonnet",
    "inherits_from": null
  },
  "claude-opus-4": {
    "aa_slug": "claude-opus-4",
    "aa_url": "https://artificialanalysis.ai/models/claude-opus-4",
    "inherits_from": null
  },
  "cursor-claude-sonnet": {
    "aa_slug": null,
    "aa_url": null,
    "inherits_from": "claude-sonnet-4-6"
  },
  "cursor-claude-opus": {
    "aa_slug": null,
    "aa_url": null,
    "inherits_from": "claude-opus-4"
  },
  "opencode-byok": {
    "aa_slug": null,
    "aa_url": null,
    "inherits_from": null
  },
  "copilot-base": {
    "aa_slug": "gpt-4o",
    "aa_url": "https://artificialanalysis.ai/models/gpt-4o",
    "inherits_from": null
  }
}
```

**Rules:**
- `null` `aa_slug` means no AA profile exists for this model (BYOK platforms, education wrappers, proprietary composites). All AA indices will be `null`.
- `inherits_from` must reference a `model_id` that has a non-null `aa_slug`. Chaining (A inherits B which inherits C) is not supported and will cause the fetch script to throw.
- Every `model_id` that appears in a provider JSON's `models[]` array must have an entry in this file. The fetch script validates this at startup.
- When adding a new model: add it to the relevant provider JSON AND to this file. Omitting it from this file will cause a validation error.

## 5. Model ID Uniqueness Convention

Model IDs must be globally unique across the entire dataset.

| Model type | Convention | Example |
|-----------|------------|---------|
| Canonical AI model (multi-platform) | Provider's canonical release ID | `gpt-4o`, `claude-sonnet-4-6`, `gemini-2-5-pro` |
| Provider-specific or wrapped model | `<provider-id>-<model-name>` | `cursor-claude-sonnet`, `copilot-base`, `copilot-xcode-preview` |
| BYOK or unknown composite | `<provider-id>-byok` or `<provider-id>-composite` | `opencode-byok`, `codeium-composite` |

The same `model_id` string must be used consistently in:
1. Provider JSON `models[].id`
2. Plan JSON `models[].model_id` (within the same provider JSON)
3. `aa-model-slugs.json` keys
4. AA snapshot file `model_id` field
5. `computed-scores.json` `scores[].model_id`

Any inconsistency in these IDs will cause the Zod validation or the score recomputation script to throw with a descriptive error identifying the mismatch.

## 6. Data Relationships

```
src/data/providers/*.json
  └─ models[].id ──────────────────────────────────► aa-model-slugs.json (key)
  └─ plans[].id ────────────────────────────────────► computed-scores.json scores[].plan_id
  └─ plans[].models[].model_id ────────────────────► (must match a models[].id in same file)

aa-model-slugs.json
  └─ <model_id>.aa_slug ───────────────────────────► AA API endpoint path
  └─ <model_id>.inherits_from ─────────────────────► another model_id in this same file
  └─ (key: model_id) ──────────────────────────────► aa-snapshots/YYYY-MM-DD.json model_id

aa-snapshots/YYYY-MM-DD.json
  └─ [].model_id ──────────────────────────────────► providers/*.json models[].id (must match)
  └─ [].inherits_from ─────────────────────────────► another model_id in same snapshot file

scripts/recompute-scores.ts (build-time)
  reads: providers/*.json + aa-snapshots/<latest>.json
  writes: computed-scores.json

computed-scores.json
  └─ scores[].plan_id ─────────────────────────────► providers/*.json plans[].id (must match)
  └─ scores[].model_id ────────────────────────────► providers/*.json models[].id (must match)
```

**Invariant:** Every `model_id` in an aa-snapshot file must have a corresponding entry in a provider JSON's `models[]` array. If a model appears in `aa-model-slugs.json` but not in any provider JSON, the validation script will flag it as an orphan.

**Invariant:** Every `plan_id` in `computed-scores.json` must match a `plans[].id` in a provider JSON. The recompute script reads plans directly from provider JSONs and only produces scores for plans it finds there.

## 7. Zod Validation Coverage

Existing Zod schemas (in `src/lib/validate.ts` or equivalent) validate the provider JSON files at build time via `npm run validate`. The following new schemas must be added to cover the new file types:

| File | Schema name | Key validations |
|------|-------------|-----------------|
| `aa-model-slugs.json` | `AAModelSlugsFileSchema` | Every key is a non-empty string; `inherits_from` if non-null must also be a key in the same object; no circular references |
| `aa-snapshots/YYYY-MM-DD.json` | `AASnapshotFileSchema` | Array of `AAIndexSnapshot`; `fetched_date` matches `YYYY-MM-DD` pattern; if `confidence === "inferred"` then `inherits_from` is required; index values are `null` or numbers between 0 and 100 |
| `aa-indices-override.json` | `AAIndicesOverrideSchema` | Same as `AASnapshotFileSchema` but every record must have `confidence === "assumed"` |
| `computed-scores.json` | `ComputedScoresFileSchema` | `computed_at` is valid ISO 8601; `formula_version` is non-empty string; every `scores[]` entry has valid `plan_id`, `model_id`, and `tier`; `value_score_normalized` is null or 0–100; `uncertainty_score` is 0–100 |

These schemas run as part of `npm run validate` and exit non-zero on any failure, blocking the build.
