## Deploy resolution (2026-06-15)

The repo was half-configured for OpenNext→Workers (`wrangler.jsonc` + `dns-fix.mjs`), but the app is 100% static (`output: "export"`, `generateStaticParams`, build-time JSON, no route handlers). Project had **never been deployed**. Switched to the correct target: static export → Cloudflare Pages.
- `next.config.ts`: added `output: "export"` + `trailingSlash: true`
- `wrangler.jsonc`: replaced OpenNext/Workers config with Pages (`pages_build_output_dir: "out"`)
- `pnpm build` → `out/` (15 static pages, 5 API JSONs) → `wrangler pages deploy out --project-name=code-smart`
- Live + verified: root 200, `/providers/<id>/` 200, all 5 `/data/api/*.json` serve valid data
- `dns-fix.mjs` no longer needed for deploy (Pages upload path avoids the LibreSSL TLS bug)
