[2026-06-16] A reported `/data/api/*.json` "edge schema-transform bug" was not real ŌĆö the RTK token-compression hook rewrote my `curl ŌĆ”json`/`cat ŌĆ”json` Bash commands into `rtk curl`/`rtk read`, which emit a type-schema summary instead of the raw body. | Root cause: RTK runs in the Bash output channel only; `head`/`python`/`jq`/WebFetch are not rewritten, so the same file looked raw via `head` but "transformed" via `curl` ŌĆö the contradiction that should have pointed at the channel, not the server. | Guard: to verify a JSON endpoint/file, use WebFetch (external egress) or bypass RTK (`command curl`, `rtk read --raw`, `head`, `jq`); never diagnose a "transform" from `curl ŌĆ”json` output alone.

[2026-06-16] Data quality checks found 4 "errors" that were false positives. pay-per-token API plans legitimately have `monthly_usd: null` (no monthly flat fee, billed per token), and BYOK/model-agnostic tools have `context_length_k: null` (variable/undisclosed context). | Root cause: Overstrict type guards treated legitimate null as missing/error data. | Guard: Pay-per-token plans can be detected via `usage_limits` containing `type: "unlimited"`; null `context_length_k` is valid for model-agnostic/undisclosed-context models. Always check domain semantics before flagging null.

[2026-06-16] 30-day test boundary failed because `date.setDate(-30)` + `toISOString()` creates a midnight UTC string that, when re-parsed by `new Date()`, yields `ageDays` slightly > 30 due to time-of-day mismatch. | Root cause: Test used 30 days boundary but UTC midnight vs local time-of-day drift pushes the comparison past the >30 threshold. | Guard: Use 29 days (unambiguously under threshold) or use fixed timestamps for date-arithmetic tests. Never rely on `setDate(-N)` with string round-trip for precision boundary testing.

[2026-06-17] `generate:value-estimates` is NOT part of the standard `pnpm build` pipeline ŌĆö it writes `public/data/model-value-estimates.json` which is read at Next.js SSG time. Running `pnpm build` without first running `pnpm generate:value-estimates` bakes stale estimates into the static HTML. | Root cause: The build script runs `generate:rankings && generate:static-api && next build`, but model-value-estimates is a separate manual/pipeline step. | Guard: Always run `pnpm generate:value-estimates` before `pnpm build` when usage-limit data changes. Add it to the `build` script or `pipeline:daily` pre-build hook.
